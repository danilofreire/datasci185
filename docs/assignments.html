Assignments

Here you will find information about the assignments for DATASCI 185. There are ten problem sets and five in-class quizzes, along with a final group project. The assignments are designed to help you practice the concepts covered in class and to develop your understanding of AI systems and their applications.

Assignment submissions are typically due on the week following their publication. You may submit via Canvas (preferred) or email (danilo.freire@emory.edu).

I encourage you to complete assignments in Jupyter notebooks (.ipynb), Word documents, or PDFs. The assignment folders contain the questions in various formats, so you can use them as templates.

Assessment Overview

  • Problem sets (10) — 50%. Short conceptual and practical tasks. Submit Jupyter notebooks (.ipynb), Word documents, or PDFs. Late submissions incur a 10% penalty per day unless authorised in advance. Collaboration for discussion is allowed but answers must be written independently; list collaborators on submission.

  • In-class quizzes (5) — 30%. Each quiz occupies a full lecture. Quizzes are open-book/open-notes and individual assessments. Discussion during quizzes is not permitted.

  • Final group project — 20%. Groups of 3–4 produce a report and a short demo video. The project involves conceptual elements and a harms assessment. More details will be provided in class.

Detailed Assignment Schedule

Module 0 — Orientation

Problem Set 1 (Due: Week 3) - Topics: Course introduction, AI overview, history of AI, transformer architecture - Covers: Lectures 1-2 - Focus: Understanding AI concepts, historical context, and current applications

Module 1 — How AI systems are designed

Problem Set 2 (Due: Week 5) - Topics: Dataset design, supervised vs unsupervised learning - Covers: Lectures 3-4 - Focus: Data quality, learning paradigms, and basic ML concepts

Problem Set 3 (Due: Week 6) - Topics: Metrics, validation, overfitting - Covers: Lecture 5 - Focus: Model evaluation, performance metrics, and validation techniques

Quiz 1 (Week 6) - Covers: Modules 0-1 (Lectures 1-5) - Format: In-class, open book, individual work

Module 2 — Language and perception

Problem Set 4 (Due: Week 8) - Topics: Language processing, embeddings, computer vision - Covers: Lectures 7-8 - Focus: Understanding how AI processes text, images, and audio

Quiz 2 (Week 10) - Covers: Module 2 (Lectures 7-9) - Format: In-class, open book, individual work

Module 3 — Retrieval, generation and pipelines

Problem Set 5 (Due: Week 10) - Topics: RAG, vector search, hallucination - Covers: Lectures 11-12 - Focus: Modern AI systems, retrieval augmentation, and reliability issues

Problem Set 6 (Due: Week 11) - Topics: Pipeline monitoring, testing, production systems - Covers: Lecture 13 - Focus: Practical aspects of deploying and maintaining AI systems

Quiz 3 (Week 11) - Covers: Module 3 (Lectures 11-13) - Format: In-class, open book, individual work

Module 4 — Data ethics and bias

Problem Set 7 (Due: Week 13) - Topics: Types of bias, fairness, documentation - Covers: Lectures 15-16 - Focus: Ethical considerations, bias identification, and governance

Problem Set 8 (Due: Week 14) - Topics: Case studies, real-world applications - Covers: Lecture 17 - Focus: Analysis of biased outcomes in various domains

Quiz 4 (Week 14) - Covers: Module 4 (Lectures 15-17) - Format: In-class, open book, individual work

Module 5 — Policy, governance and social impact

Problem Set 9 (Due: Week 16) - Topics: AI regulation, privacy, data protection - Covers: Lectures 19-20 - Focus: Legal frameworks, policy implications, and privacy concerns

Quiz 5 (Week 16) - Covers: Module 5 (Lectures 19-21) - Format: In-class, open book, individual work

Module 6 — Applications, limits and projects

Problem Set 10 (Due: Week 17) - Topics: AI applications, misinformation, societal impact - Covers: Lectures 23-24 - Focus: Real-world applications and their consequences

Final Group Project

Due: End of Semester
Format: Groups of 3-4 students
Components: - Written report on an AI application - Short demo video - Harms assessment - Conceptual analysis

Key Elements: - Design a small, realistic plan for an AI application - Include data needs, evaluation strategy, and harm-mitigation approach - Reflect on ethical, legal, and social questions - Demonstrate understanding of course concepts

More detailed instructions for the final project will be provided in class during the second half of the semester.

Submission Guidelines

File Formats

  • Preferred: Jupyter notebooks (.ipynb) with clear documentation
  • Alternative: Word documents or PDFs with code screenshots
  • Final Project: Report (PDF) + demo video (MP4)

Collaboration Policy

  • Discussion with classmates is encouraged for conceptual understanding
  • All written work must be your own
  • List all collaborators on your submission
  • Declare any use of AI tools (ChatGPT, Claude, etc.)

Late Policy

  • 10% penalty per day for late submissions
  • Extensions available only with prior approval
  • Medical emergencies require documentation

Academic Integrity

  • All AI use must be declared on submissions
  • You are responsible for the correctness of your work
  • Proper attribution required for all sources, including AI-generated content
  • Emory Honour Code applies to all assessments

Getting Help

If you need assistance with assignments: 1. Review lecture materials and readings 2. Consult with classmates (for discussion, not copying) 3. Attend office hours or schedule an appointment 4. Post questions in the course discussion forum 5. Email the instructor for specific concerns

Remember that the purpose of these assignments is to help you learn and apply the concepts from the course. Don’t hesitate to ask for help when needed!
